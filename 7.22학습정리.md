# Module & Control of Flow(모듈과 제어문)
---------
## 모듈
한 파일로 묶인 변수 함수 모음 (파이썬 실습 파일 하나하나가 모듈)
(Ex) math 내장 모듈

### import 문
같은 이름 함수가 여러 모듈 안에 있을 때 충돌 방지
.(dot) 연산자 - 점 왼쪽 객체에서 오른쪽 이름을찾아라 (math.pi)형식
충돌 시에는 뒤에 온 import 함수가 우선

### from 절
import는 단독 문장 사용 가능. from은 import에 의존적인 절 하나.
- 필요한 것만 특정해서 가져오는 방식 (앞에 변수함수 최초 이후안써도됨)
import : 모듈 이름짧음. 어디서 온 함수인지 드러낼 때 (math.pi)
from : 특정함수/변수 반복호출 시

#### as 키워드
as키워드 사용 별칭 부여 - 이름 충돌 해결

- import* 는 사용하지 않는다.

### 사용자 정의 모듈
.py 파일 생성을 통해 모듈 생성 -> 함수 정의 후 그 안에 있는 함수를 모듈 이름으로 참조
실습파일 my_math, sample 참조

### 패키지
연관된 모듈들을 하나의 디렉토리에 모아 둔 것
패키지 안에 패키지 들어갈 수 있음. 모듈의 효율적 관리
실습파일 sample.py #직접만든 패키지 사용하기 참조

### 라이브러리
패키지를 관리하는 공간 표준 라이브러리/외부 패키지
- pip : 파이썬 패키지 관리 시스템(외부패키지 설치 도움)

#### 사용자 정의 라이브러리 활용(실습파일 04_requests 참조)
[동작 흐름] 브라우저 주소창에 URL을 입력하는 것과 같은 일을 코드로 수행
    1. requests.get(url)  : 해당 주소로 데이터를 요청
    2. .json()            : 응답으로 받은 JSON 문자열을 딕셔너리/리스트로 변환
    3. print(response)    : 변환된 파이썬 자료구조를 출력
pprint 함수 : 자료형 자동 줄바꿈 및 정렬하여 정리 출력 도움

## 제어문
코드의 실행 흐름을 제어.
조건에 따라 코드블록 실행 / 반복실행
조건문 : if, elif, else
반복문 : for, while

### 조건문
조건에 따라 True일때 실행 혹은 건너뛰기
if는 단독, elif/else 단독사용 불가
조건을 순서대로 읽기때문에 *조건문의 순서 중요함*
if : 조건을 표현식으로 작성 - 중첩 가능
elif : 추가로 조건 필요 시
else : 조건 모두 불만족 시 실행
if 구문 여러개 vs elif 사용 시 같은 조건 넣어도 결과 달라짐

### 반복문
주어진 코드블록을 여러 번 반복해서 실행하는 구문

### for 구문
반복가능한(iterable) 객체 요소 반복
반복 횟수가 정해져있음, 반복변수는 단수형(반복은 여러번하지만
하나를출력 혹은 참조하는 과정이므로)

- 특정 숫자만큼 반복할 때 range 사용
- 인덱스 순회 : 원본 리스트 수정해야할 때 
range(len(numbers))로 0 ~ 마지막 인덱스를 만들어 사용
- dict : 딕셔너리를 그냥 순회하면 '키'가 꺼내짐 (값이 아님)
값이 필요하면 my_dict[key]로 다시 접근

- 중첩 반복문 : 반복 안의 반복
바깥 1회당 안쪽이 전부 돌고 나서 바깥이 다음으로 넘어감
- 중첩 리스트 순회 - 한 겹 벗길 것인가, 두 겹 벗길 것인가
1: 한 번만 순회 => 안쪽 '리스트' 자체가 꺼내짐 (['A', 'B'])
2: 두 번 순회 => 리스트 안의 '요소'까지 꺼내짐 (A, B, c, d)
-> Im 테스트 2차원리스트 순회 수준

### while 구문
while 조건이 참인동안 반복 = False가 될때까지 반복(수렴)
for문과 달리 반복 횟수가 미리 정해져 있지 않음
- 반드시 필요한 3요소
① 시작 값 (count = 1)
② 반복 조건 (count <= 3)
③ 조건을 변화시키는 코드 (count = count + 1)
-  반드시 종료 조건이 필요

#### for, while 비교
for:
iteralble 요소 순회
반복 횟수 정해진 경우 유용(시퀀스 형식 처리, range()로 반복작업)
while:
조건 참인동안 반복
반복횟수 불명확, 조건부 반복 종료 시 유용

### 반복제어
때때로 반복문 일부만 실행하는게 필요할 때

- break 
해당 키워드 이후 남은코드 무시하고 즉시종료.(조건 명확할 시)
어디 들어있는지 확인
- continue 
다음 코드 무시하고 다음 반복 수행(고의 누락)
- pass
아무 동작도 하지 않음.
코드의 틀 유지, 나중에 내용 채우기 위한 용도. (코드 비우기 대신)
함수, 조건문에도 사용 가능

### 내장함수 map  map(function,iterable)
iterable의 모든 요소에 function을 적용하고, map으로 묵어서 반환
- input 과 함께 쓰는 패턴 list(map(int, input().split()))
- lambda와 함께 사용 list(map(lambda x: x**2, numbers))
map의 첫 인자는 '함수 자체' => 괄호 없이 이름만 전달 (square)
labssafy map 입문편 프로젝트 git 참고

#### zip함수 (*iterables)
데이터를 묶어서 tuple로 만든 뒤 모아서 zip object로 반환.
갯수 안맞으면 함수실행 무시됨(행렬과 연계되는 구조)

### for-else 구문
for이 break를 만나 중단되지않고, 정상적 완료 시 else 실행
조건부 반복 실행

### enumerate  (iterable, start=0)
각 요소 인덱스와 값을 함께 반환하는 내장함수 (인덱스, 요소)
start 원점 가변 -> start=1 가능
