# git branch

대괄호나 부등호기호표는 안의 내용이변경될 수 있음 <change> [가능]
브랜치는 병합이 되면 역할이 끝난다.
HEAD -> 우리가 보고있는 위치(브랜치/commit). switch는 헤드를 이동하는 명령어
브랜치에서 각각의 브랜치는 병합 전까지 각각의 디렉토리 - switch 전 버전관리 필수
진행방향은 시계열. but 과거 발생한 일에 종속되는 개념으로 commit이 저장되기때문에 화살표는 왼쪽
* 위에 파일 코드 작성란과(explorer탭), 터미널 란 채팅 옮겨가며 치는법(마우스없이)

fastfoward - 브랜치가 복사는 되었지만 실제 기술적 병합이 된것은 아님
말그대로 다음 커밋 단계로 빨리감기해서 이동
vim - 마우스 입력을 못하는 언어(프로그램 조상격)
입력모드/명령모드 2가지
:(콜론) 입력시 명령모드 진입

# 기획과 설계
MVP - 최소 포함 품목(초안을 잡을 때) 
WBS - 작업 분류 구조(프로젝트 목표를 달성하기 위해 수행할 전체 업무 범위를 계층적으로 세분화하는 도구)
간트 차트(Gantt Chart)는 프로젝트 일정과 작업을 시간순으로 시각화한 바(Bar) 형태의 도구
