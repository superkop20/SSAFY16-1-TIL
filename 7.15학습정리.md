# forking workflow
- 중앙원본저장소(upstream)에 직접 Push 불가
fork -> github repositary 복사 
fork - clone - 이름바꿔 remote - 브랜치 작업 - push - PR생성 및 원본저장소 전송 - merge 되면 master브랜치
remote 는 원격저장소를 연결하는 명령
