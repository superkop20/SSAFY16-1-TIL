### 딕셔너리형
- 키와 값을 짝지어 저장하는 구조 {key : value}

### 딕셔너리 메서드
D.get(k/k,v), D.keys, D.values, D.items, D.pop(k/k,v)
- get(k/k,v) : 키 에 연결된 값 반환 Or 기본값 v반환
- keys : 딕셔너리 키 객체 반환 - 리스트로 출력
- values : 값 객체만 반환
- items : 키/값 쌍을 반환 - 튜플로 출력
- setdefault : 조회, 만들어넣기. update : 추가, 덮어쓰기
- pop : 키 제거후 연결된 값 반환

#### defaultdict (딕셔너리의 확장) {<class 'int'>, (...)}
딕셔너리에 '존재하지않는 키' 조회할때 자동기본값을 생성해주는 자료구조
기본 vs default : 기존 - 매번 키존재여부 확인
개선 - 초기화 코드 없이 없는 키를 자동으로 만들어주는 딕셔너리
[그래프, 분류]문제에서 강력하게 활용.
setdefault - 호출마다 매번 기본값, defaultdict - 한번만 하면됨

### 세트 메서드
- .add(x) : 세트 항목 하나 추가
- .update(iterable) : 여러항목 한번에 추가
- .clear : 전체 제거 - 빈세트는 set()
- .remove(x) : 항목 x 제거, x 없으면 키에러
- .pop : 임의항목 반환, 제거
- .discard(x) : 제거, 키에러 없음

### 해시테이블
- 키와 값을 짝지어 저장하는 자료구조
1. 키를 해시함수를 통해 해시값으로 변환
2. 변환된 값을 인덱스 삼아 데이터 저장/탐색

### 해시 
'임의'의 크기를 가진 데이터를 '고정'크기의 고유값으로 변환하는 것
- 변환을 수행하는 것이 해시함수
#### 해시함수(해시 알고리즘)
- 임의길이 데이터를 입력받아 고정길이(정수)로 변환해주는 함수
(정수) = 해시 값. 

### set, dict 해시테이블 관계
set : 각 요소를 해시함수 변환, 해시값 맞춘 버킷 저장
순서X, 버킷위치(인덱스)가 요소위치결정 -> 순서를 보장하지 않음.
dict : 키 - 해시함수 - 해시 값 - 해시테이블
set와 달리 삽입순서는 유지 -> 추가한 순서대로 출력

set pop 정수
- 정수 값은 해시값이 자신과동일하거나 단순계산으로 고정됨
정수또한 임의이므로 순서는 없음. pop 예측 할 수 없음.
set pop 문자열
- 문자열은 해시계산 시 해시난수화 적용. 실행마다 랜덤
- 난수시드 : 프로세스 새로 시작될때마다 달라짐 - 함수 불변, 시드값 실행마다 달라짐
동일한 데이터라도 해시값이 달라져 버킷배치 바뀜

#### 파이썬에서 해시함수 
정수 : 같은 정수는 항상 같은 해시값을 가짐
Ex - hash(1) 여러번 호출해도 같은 값
문자열 : 문자열 해시시, 난수 시드 달라질 수 있음(보안상 이유)
Ex - hash('a') 값 실행마다 다름
jpynb : 주피터 노트북 파일(해시함수 테스트 파일)

#### hashable
- hash()함수에 넣어 해시 값을 구할 수 있는 객체
- 대부분 불변타입 해시 가능, 가변형객체 기본적으로 해시 불가능(불변성)
set, dict 키에 hashable 객체만 저장가능
불변객체 - 항상 같은 해시값(생성 후 값 변경이 불가능)
일반적으로 내장 자료형기준 불변이어야 해시 가능
